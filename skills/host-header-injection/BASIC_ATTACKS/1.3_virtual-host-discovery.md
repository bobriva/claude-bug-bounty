# 1.3 Virtual Host Discovery

## Goal

Discover hidden applications/virtual hosts by fuzzing Host header values.

---

## What are Virtual Hosts?

Single server hosts multiple applications based on Host header:

```
Host: target.com    → Main application
Host: admin.target.com → Admin panel
Host: api.target.com   → API
Host: staging.internal → Staging environment
```

Server routing logic:
```
if Host == "target.com":
    serve main_app
elif Host == "admin.target.com":
    serve admin_panel
elif Host == "internal":
    serve internal_tools
```

---

## Common Virtual Host Names

**Public subdomains:**
- www, mail, ftp, blog, shop, cdn

**Admin/Dev:**
- admin, administrator, superadmin
- dev, devel, development
- staging, pre-prod, test
- beta, sandbox, demo

**Internal:**
- internal, private, secret
- backend, api, gateway
- database, cache, queue
- monitoring, logging, metrics

**Cloud-specific:**
- metadata, cloud, aws, gcp, azure
- admin-console, dashboard

## Discovery Methods

### Method 1: Manual Fuzzing
```bash
# Try common names
for host in admin dev staging internal beta metadata; do
  curl -H "Host: $host.target.com" https://target.com/ 2>/dev/null | grep -q "<title>" && echo "Found: $host"
done
```

### Method 2: FFUF Fuzzing
```bash
ffuf -w wordlist.txt -H "Host: FUZZ.target.com" \
     -u https://target.com/ \
     -fw 10  # Filter by word count
```

### Method 3: Burp Intruder
1. Use Host: FUZZ in Intruder
2. Payload: wordlist.txt
3. Filter by: status code changes, content length, unique titles

---

## Indicators of Discovery

**Signs you found a vhost:**
- Different status codes (200 vs 403)
- Different content length
- Unique page titles
- New application names
- Different error pages
- Different technologies
- Unique response headers

## Common Discoveries

1. **admin.target.com** → Admin panel (SQL injection possible)
2. **api.target.com** → API endpoint (missing auth checks)
3. **staging.target.com** → Debug features enabled
4. **internal.target.com** → Internal tools
5. **dev.target.com** → Development version (hardcoded credentials)
