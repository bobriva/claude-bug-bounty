# Business Logic Skill

Use this skill when:

- Traditional vulnerability testing yields little value
- Financial workflows exist
- Multi-step workflows exist
- Approval systems exist
- Rewards systems exist
- Discounts exist
- Loyalty systems exist
- Payment systems exist

## Objectives

1. Understand intended business rules
2. Identify hidden assumptions
3. Break workflow expectations
4. Manipulate state transitions
5. Abuse trust relationships
6. Discover unintended outcomes

## Core Principle

Do not ask:

"Is this vulnerable?"

Ask:

"What assumptions did developers make?"
