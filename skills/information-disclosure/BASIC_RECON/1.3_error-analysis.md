# 1.3 Error Analysis

## Goal

Analyze error messages and stack traces to discover system information and exploit paths.

---

## Stack Traces

Error pages often reveal full stack traces:

**Java Exception:**
```
java.sql.SQLException: Access denied for user 'root'@'localhost'
at com.mysql.jdbc.SQLError.createSQLException
```

**PHP Error:**
```
Fatal error: Uncaught PDOException: SQLSTATE[HY000] [1045] Access denied for user 'admin'@'localhost'
in /var/www/html/db.php:12
```

**Node.js Error:**
```
Error: Cannot find module 'express'
at Function.Module._load (internal/modules/require.js:214:12)
at /home/user/app/index.js:5:3
```

**Information leaked:**
- Framework names & versions (vulnerable?)
- Database names & usernames
- File paths (/var/www/html/, /home/user/)
- Function names (logic hints)

---

## Framework Identification

Stack traces reveal:
- **Laravel:** `Laravel/framework`
- **Express:** Node.js path `/node_modules/`
- **Flask:** Python path `/site-packages/`
- **Django:** Python Django imports
- **Spring:** Java package names

**Exploitation:** Search for CVEs for specific versions!

---

## Database Information

Error messages may reveal:
- Database type (MySQL, PostgreSQL, SQL Server)
- Username (`root@localhost`)
- Partial password hints
- Database names

**Example:**
```
SQLSTATE[HY000] [1045] Access denied for user 'root'@'localhost'
```

Tells you: User is `root`, host is `localhost` (local database)

---

## File Path Disclosure

Stack traces reveal server structure:

```
/var/www/html/index.php
/home/user/app/routes.php
/opt/application/core/database.php
```

**Information:**
- Web root location
- Application structure
- File names hint at function
- Potential RCE paths

## Triggering Errors

Try these inputs:

```
Invalid queries: `'; DROP TABLE--`
Type errors: Pass string where int expected
Null pointers: Pass null to required field
File access: Try ../../../../etc/passwd
SQL injection: Standard payloads
```
